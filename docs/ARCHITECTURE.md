# 架构状态

新一轮 Nora 架构尚未确认。本文件只记录当前治理边界，不预先承诺技术栈或物理目录。

## 已确认原则

- 系统服务于可解释、可追溯的求职决策。
- 业务事实、模型推断和用户建议必须分离。
- Agent 只能编排应用用例，不能直接拥有业务事实或绕过授权边界。
- 外部输入默认不可信；外部写默认关闭并要求审批、幂等和审计。
- 数据所有权、进程边界、依赖方向和技术选型必须由 Architecture Issue 决定。

## 尚待决策

- 领域上下文和首个纵向切片；
- API、Worker、Agent 与浏览器进程边界；
- 关系数据、向量数据、对象和长期记忆的所有权；
- 模型、Embedding、检索与任务框架；
- 身份、授权、隐私、审计和部署模型；
- 本地开发、测试和生产质量门禁。

在这些决策通过 Issue 与 PR 验收前，不创建应用骨架、占位业务目录或运行时依赖。
